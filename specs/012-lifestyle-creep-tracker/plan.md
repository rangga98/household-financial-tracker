## Summary

**Primary Requirement**: Build an analytics module that compares income growth vs expense growth over time periods (3/6/12 months or custom), issuing visual warnings when expenses outpace income (lifestyle creep detection).

**Technical Approach**: Pure derived-state architecture aggregating existing `transactions` table data via Supabase queries. UI built with Tremor LineChart for trend visualization and Shadcn/ui Alert/Select for warnings and filtering. TDD-first development with 100% test coverage on `calculateGrowthPercentage()` utility using Vitest. No database migrations required.

## Technical Context

**Language/Version**: TypeScript 5.x with strict mode enabled
**Primary Dependencies**: Next.js 14+ (App Router), React 18+, Tremor (data visualization), Shadcn/ui (components), Supabase (PostgreSQL + Auth), Tailwind CSS
**Storage**: Supabase PostgreSQL (existing `transactions` table only - NO NEW TABLES, pure derived state)
**Testing**: Vitest + React Testing Library (RTL) - TDD mandatory per Constitution III
**Target Platform**: Web (mobile-first responsive: iPhone/Android → tablet Bento grid → desktop multi-column)
**Project Type**: Web application (Next.js full-stack)
**Performance Goals**: <3 second load time for comparison view (per SC-001), 100% test coverage for math utilities (per SC-006)
**Constraints**: Zero-cost deployment (Vercel + Supabase free tier), YAGNI (derived state only, no DB migrations), 44x44px minimum touch targets
**Scale/Scope**: Single analytics feature module, household-level data (multi-user read access to same transaction ledger)
